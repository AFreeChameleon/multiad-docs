Introduction

Multask is a daemoniser built to simplify & organise your developer environment.

It was designed to help organise and tidy up projects with lots of services which normally would lead to many terminal tabs open and very little monitoring.

IMAGE OF MULTASK TABLE HERE

Supports:

  • Windows
  • MacOS
  • Linux (tested on Debian & Arch)
  • FreeBSD

Getting Started

Creating your first process

After installing the program run:

> mlt create "ping 8.8.8.8"

The command's output won't be shown in the console, but instead the message:

Success Process 0 created.

0 is the ID assigned to the process and will be used when you want to stop or restart the process.

View the process's output

Nothing will be shown in the console because the process has been daemonized, and all output from the process will be stored in a logs file.

To view that log file run:

> mlt logs 0

List all processes

To see all processes you've created, run:

> mlt ls

It will output a table like this:

┌────┬─────────────────┬───────┬─────────┬─────────┬─────┬─────────┐ │ id │ command │ pid │ status │ memory │ cpu │ runtime │ ├────┼─────────────────┼───────┼─────────┼─────────┼─────┼─────────┤ │ 0 │ ping google.com │ 12502 │ Running │ 9.9 MiB │ 0 │ 16 │ └────┴─────────────────┴───────┴─────────┴─────────┴─────┴─────────┘
  • id - how you'll be referencing this process in other commands.
  • command - what command was run.
  • pid - process id in the OS.
  • status - status of the command, either 'Running' or 'Stopped'.
  • memory - amount of memory the process is using.
  • cpu - percentage of CPU the process is using.
  • runtime - how long this command has been running for.

Process operations

To stop the process, run:

> mlt stop 0

To start the process, run:

> mlt start 0

To restart the process, run:

> mlt restart 0

To delete the process and all logs, run:

> mlt delete 0

If multask isn't working, you can run:

> mlt health

To see what's wrong with it. This is mainly for debugging purposes.

Bugs

How to report

This is a very early alpha and while this has been tested on a few environments, issues on other environments (especially using shells other than bash and zsh) are untested and may have some breaking issues.

So please report any problems you run into (or feature requests) here.
